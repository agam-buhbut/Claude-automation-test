# Client provisioning

Two clients connect to the server simultaneously: an Android phone and a second
Debian machine. Both use the **same** AmneziaWG obfuscation parameters as the
server (already baked into their generated configs).

Configs are generated by `server/gen-config.py` into `secrets/` (git-ignored):
`client-android.conf`, `client-debian.conf`. Regenerate with a real public
endpoint before delivery:

```bash
python3 server/gen-config.py vpn.example.com   # or a public IP
```

## Android (Amnezia app)

The **Amnezia VPN** app speaks AmneziaWG natively.

- Install from Google Play, F-Droid (`org.amnezia.vpn`), or the signed APK at
  <https://github.com/amnezia-vpn/amnezia-client/releases>.
- Render the config as a QR and scan it:

  ```bash
  clients/make-qr.sh --show      # prints a scannable QR in the terminal
  ```

  In the app: **+ → Import → Scan QR code**. (The QR encodes the client private
  key — keep it private.)

## Debian (second machine)

Deliver `secrets/client-debian.conf` to that machine out-of-band (e.g. `scp`),
then run the self-contained installer **on the client**:

```bash
sudo clients/debian/install-client.sh client-debian.conf
```

It installs dependencies, builds AmneziaWG (userspace), installs the config,
brings the tunnel up via systemd, and verifies the handshake + egress IP.

## Reachability

The server is behind NAT, so clients reach it via either:

- **IPv4** — forward `UDP/443` on the router to the server's LAN IP
  (`192.168.1.3`); set the client `Endpoint` to the router's public IPv4 (use
  dynamic DNS if it is not static), **or**
- **IPv6** — the server has a global IPv6 address; allow inbound `UDP/443` to it
  in the router's IPv6 firewall and set the client `Endpoint` to that address.
  (No NAT/port-forward needed for IPv6.)

The `Endpoint` is the only field that depends on this choice; everything else is
identical.
